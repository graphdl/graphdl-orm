# Domain DO + DO Adapter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract the metamodel from the monolithic `GraphDLDB` into a `DomainDB` Durable Object, and create a DO adapter so the existing `ingestProject` step functions work with Domain DO RPC.

**Architecture:** `DomainDB` extends `DurableObject`, holds metamodel tables (nouns, readings, constraints, etc.) in its own SQLite. The DO adapter implements the same `findInCollection`/`createInCollection`/`updateInCollection` interface that the step functions expect, but routes calls to the Domain DO. This lets all 504+ existing tests continue to work.

**Tech Stack:** TypeScript, Cloudflare Workers Durable Objects, SQLite, Vitest

**Spec:** `docs/superpowers/specs/2026-03-20-do-per-entity-design.md`

```typescript
// src/domain-do.ts
import { BOOTSTRAP_DDL } from './schema/bootstrap'

export const METAMODEL_TABLES = [
  'nouns', 'graph_schemas', 'readings', 'roles',
  'constraints', 'constraint_spans',
  'state_machine_definitions', 'statuses', 'transitions', 'guards',
  'event_types', 'verbs', 'functions', 'streams',
  'generators',
]

export function initDomainSchema(sql: SqlLike): void {
  for (const statement of BOOTSTRAP_DDL) {
    // Only execute DDL for metamodel tables
    const tableMatch = statement.match(/(?:CREATE TABLE|CREATE INDEX).*?(?:IF NOT EXISTS\s+)?(\w+)/)
    if (tableMatch) {
      const tableName = tableMatch[1]
      // For indexes, extract the target table name
      const indexTarget = statement.match(/ON\s+(\w+)/)
      const relevantTable = indexTarget ? indexTarget[1] : tableName
      if (METAMODEL_TABLES.includes(relevantTable)) {
        try { sql.exec(statement) } catch {}
      }
    }
  }
}
```

```typescript
// src/do-adapter.ts

/**
 * Creates a GraphDLDB-compatible adapter that routes collection operations
 * to a DomainDB Durable Object via RPC.
 *
 * This lets the existing step functions (ingestNouns, ingestReadings, etc.)
 * work with Domain DOs without changing their signatures.
 */
export function createDomainAdapter(domainDO: DomainDBStub): GraphDLDBLike {
  return {
    findInCollection: (collection, where, opts) => domainDO.findInCollection(collection, where, opts),
    createInCollection: (collection, data) => domainDO.createInCollection(collection, data),
    updateInCollection: (collection, id, updates) => domainDO.updateInCollection(collection, id, updates),
    applySchema: () => domainDO.applySchema(),
  }
}

export interface DomainDBStub {
  findInCollection(collection: string, where: any, opts?: any): Promise<{ docs: any[]; totalDocs: number }>
  createInCollection(collection: string, data: any): Promise<any>
  updateInCollection(collection: string, id: string, updates: any): Promise<any>
  applySchema(): Promise<{ tableMap: Record<string, string>; fieldMap: Record<string, Record<string, string>> }>
}

export interface GraphDLDBLike {
  findInCollection(collection: string, where: any, opts?: any): Promise<{ docs: any[]; totalDocs: number }>
  createInCollection(collection: string, data: any): Promise<any>
  updateInCollection(collection: string, id: string, updates: any): Promise<any>
  applySchema(): Promise<{ tableMap: Record<string, string>; fieldMap: Record<string, Record<string, string>> }>
}
```
