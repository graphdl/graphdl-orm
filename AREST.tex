\title{\textbf{Compiling Facts into Applications}}

\author{Samuel Lippert\\Drivly Inc}

\begin{abstract}
Domain knowledge enters software as natural language requirements, is re-encoded as schemas, validation logic, state machines, and API routes, and drifts out of sync in production. The informal layer between specification and execution is where bugs live and where automated agents hallucinate. We show that a single set of FORML\,2 declarations can define a database schema, constraint rules, state machine workflows, and a REST API with hypermedia navigation, producing a complete application from domain facts alone. We formalize this as AREST, a Backus AST system~\cite{backus78} whose definitions are compiled FORML\,2 readings~\cite{halpin01} and whose state is Codd's named set~\cite{codd70} of ORM\,2 elementary facts. REST endpoints are applications of a single system function. Ingesting new readings is self-modification. Five theorems establish that the grammar is unambiguous, that specifications and executables are the same object, that state transfer is complete, that HATEOAS links are projections of the population, and that every value in the API is derived from facts. Together, they eliminate the informal layer entirely.
\end{abstract}

\section{Introduction}

Software is increasingly written and consumed by agents. An LLM writing code produces a specification it cannot verify against the resulting executable; an LLM reading an API navigates documentation that may have drifted from the endpoints. In both cases the informal layer between specification and execution is where the agent hallucinates. The rest of this paper shows why that layer need not exist.

Domain knowledge begins in someone's head. It is re-encoded as schemas, validation logic, state machines, and API routes. When these are distinct artifacts, their equivalence is not preserved by construction.

AREST identifies what was never separate to begin with. A FORML\,2 reading is simultaneously a relation schema, a constraint specification, a resource definition, and an FFP object. The compiler does not translate between representations. It recognizes a single object that already occupies all four roles.

\section{Terminology}

Three terms carry overloaded abbreviations. \emph{AST} follows Backus's usage (Applicative State Transition, 1978), not the compiler term. \emph{ORM} follows Halpin's usage (Object-Role Modeling, 2008), not the persistence pattern. \emph{FP} follows Backus's usage (Function-level Programming), not the broader tradition. \emph{FFP} is Backus's extension of FP with named definitions and a representation function $\rho$ that maps objects to the functions they denote~\cite[Sec.\,13]{backus78}.

\section{Definition}

\begin{definition}[AREST]\label{def:arest}
An \emph{Applicative REpresentational State Transfer} system is an AST system~\cite[Sec.\,14]{backus78} whose state $D$ satisfies:
\begin{itemize}[nosep]
\item The $\mathit{FILE}$ cell contains a population $P = \uparrow\!\mathit{FILE}\!:\!D$, a named set of relations~\cite[Sec.\,1.6]{codd70} whose elements are ORM\,2 elementary facts~\cite[Ch.\,3]{halpin01}
\item The $\mathit{DEFS}$ cells contain FFP objects~\cite[Sec.\,13.2]{backus78}: readings compiled from FORML\,2 declarations~\cite{halpin01}, and functions registered by the runtime
\end{itemize}
The \emph{schema} $S$ is the set of compiled objects in $\mathit{DEFS}$: fact type constructors, constraint objects, derivation rules, and state machine definitions. The representation function $\rho$~\cite[Sec.\,13.3.2]{backus78} maps an object to the function it represents. Given a fact type, $\rho$ looks it up in $\mathit{DEFS}$ and returns the functional form. The system function $\mathit{SYSTEM}\!:\!x = \langle o, D'\rangle$~\cite[Sec.\,14.3.1]{backus78} is defined by the definitions in $D$. REST endpoints are applications of $\mathit{SYSTEM}$ with distinct inputs. A reference implementation is available~\cite{arest}.
\end{definition}

\section{Example}\label{sec:example}

An order domain is declared in FORML\,2:

\begin{lstlisting}
Order(.OrderId) is an entity type.
Customer(.Name) is an entity type.
Order was placed by Customer.
  Each Order was placed by exactly one
  Customer.
Status 'In Cart' is initial in
  State Machine Definition 'Order'.
Transition 'place' is from
  Status 'In Cart'.
Transition 'place' is to
  Status 'Placed'.
Transition 'ship' is from
  Status 'Placed'.
Transition 'ship' is to
  Status 'Shipped'.
\end{lstlisting}

\noindent These nine readings compile to a database table, a foreign key, a uniqueness constraint, and a three-state machine, which is everything needed to run an order pipeline. Consider the constraint ``Each Order was placed by exactly one Customer.'' The compiler recognizes this as a uniqueness restriction on the $\mathit{placedBy}$ role and emits:
\begin{align*}
(\rho\,\mathit{reading})\!:\!P_{\mathit{placedBy}} = \mathit{Filter}(\mathit{count}(s_1) > 1) : P_{\mathit{placedBy}}
\end{align*}
The result is the set of Orders with more than one Customer in $\mathit{placedBy}$. An empty set means the constraint holds.

\texttt{POST /orders \{"customer":"acme"\}} applies $\mathit{SYSTEM}\!:\!x$ where the addressed entity is $\mathit{orders}$ and the operation is $\mathit{create}$. The pipeline produces:
\begin{align*}
P' = P \cup \{&\mathit{Order}(\texttt{ord\text{-}1}),\;
\mathit{placedBy}(\texttt{ord\text{-}1}, \texttt{acme}),\\
&\mathit{SM}(\texttt{ord\text{-}1}, \texttt{In Cart})\}
\end{align*}

The response includes links derived from $P'$:
\begin{lstlisting}
{ "id": "ord-1", "customer": "acme",
  "status": "In Cart",
  "_links": {
    "place": {
      "href": "/orders/ord-1/transition",
      "method": "POST" } } }
\end{lstlisting}

\noindent Following ``place'' advances the state machine. The new representation contains $\{\mathit{ship}\}$, not $\{\mathit{place}\}$, because the status is now Placed. If a second command assigns a different Customer to the same Order, the restriction produces a non-empty violation set and the command is rejected. The violation is returned as the original reading.

The constraint, the state machine, the API response, and the HATEOAS links above are different views of the same data. The mechanism that makes this possible is metacomposition. A fact type~\cite[Ch.\,3]{halpin01} is a constructor of roles~\cite[Sec.\,11.2.4]{backus78}. Populate the roles with instances and you get a fact: $\langle\mathit{ft}, o_1, \ldots, o_n\rangle$. Metacomposition~\cite[Sec.\,13.3.2]{backus78} resolves a fact by looking up its fact type:
\begin{align}
(\rho\langle x_1, \ldots, x_n\rangle)\!:\!y = (\rho\,x_1)\!:\!\langle\langle x_1, \ldots, x_n\rangle, y\rangle \label{eq:metacomp}
\end{align}
The fact is a function: $\lambda f.\, f(o_1, \ldots, o_n)$, accepting an operation and applying it to its own objects. The constraint reading, the state machine transition, and the API response above are all $\rho$-applications over the same facts. The sections that follow show why they cannot be anything else.

\section{The System Function}

Each input $x$ addresses an entity and carries an operation. The system function routes the input to the addressed entity and passes the operation as the operand:
\begin{align}
\mathit{SYSTEM}\!:\!x &= (\rho\,(\uparrow\!\mathit{entity}(x)\!:\!D))\!:\!\uparrow\!\mathit{op}(x) \label{eq:sys}
\end{align}
The entity is fetched from $D$. $\rho$ resolves it to a functional form. The operation is applied as the operand. The entity handles the dispatch, not the system function. New operations are registered in $\mathit{DEFS}$ without modifying any entity, and new entity types are added without modifying any operation.

\smallskip\noindent\textbf{Remark} (Specialization of Backus's subsystem). Backus's subsystem~\cite[Sec.\,14.4.2]{backus78} dispatches on a $\mathit{KEY}$ cell with five clauses. AREST collapses them into a single $\rho$-dispatch~(\ref{eq:sys}): the entity determines the functional form, and the HTTP method determines the operation class. Self-modification is the same mechanism: the addressed entity is $\mathit{DEFS}$ and the operation is $\mathit{compile} \circ \mathit{parse}$. The new FFP objects are stored via $\downarrow\!\mathit{DEFS}$, and subsequent applications of $\mathit{SYSTEM}$ evaluate the new definitions.

\subsection{Commands}

$\mu$ denotes evaluation~\cite[Sec.\,14.3.1]{backus78}. Each application is an AST transition: the state before, the state after, nothing else changes.

A \textsc{Create} command:
\begin{align}
\mathit{create} &= \mathit{emit} \circ \mathit{validate} \circ \mathit{derive} \circ \mathit{resolve} \label{eq:create}
\end{align}
$\mathit{resolve}$ determines identity from the reference scheme~\cite[Ch.\,5]{halpin01}. $\mathit{derive}$ forward-chains derivation rules to the least fixed point. $\mathit{validate}$ applies every constraint as a restriction over the population, producing violation sets. $\mathit{emit}$ always runs. An alethic violation rejects ($D' = D$). A deontic violation warns but succeeds ($D' = D''$). The violation message is the original reading (Corollary~\ref{cor:verbalize}).

A state machine processes events chronologically:
\begin{align}
\mathit{machine}(s_0, E) &= \mathit{foldl}\;\mathit{transition}\;s_0\;E \label{eq:sm} \\
\mathit{transition}(s, e) &= (p \to \bar{s'} ;\; \bar{s})\!:\!\langle s, e \rangle \label{eq:trans}
\end{align}
The transition function is a chain of conditions~\cite[Sec.\,11.2.4]{backus78}: $p$ tests whether $\langle s, e\rangle$ matches a declared transition, $f$ returns the target state, $g$ tries the next or returns $\bar{s}$ as fallback. The fold is pure replay. Applying the event stream produces the current state without side effects.

\smallskip\noindent\textbf{Facts as events.} Creating a fact fires an event. The event type is the fact type; the event arguments are the role bindings. A transition that declares its trigger as a fact type fires automatically when that fact enters $P$. During $\mathit{create}$~(\ref{eq:create}), $\mathit{resolve}$ and $\mathit{derive}$ push facts into $P$; each produces an event that the machine fold~(\ref{eq:sm}) consumes. An entity whose facts satisfy all outgoing transition guards advances from its initial status to its terminal status in a single $\mathit{SYSTEM}$ call. External events from a message queue, a webhook, or a peer enter the same stream without requiring a local fact. The machine does not distinguish between fact-fired and externally fired events; both are elements of $E$.

Deletion is a transition to a terminal state. The entity reaches a state with no outgoing transitions and is excluded from query results by restriction.

\subsection{Platform Binding}

The runtime registers functions into $\mathit{DEFS}$ via $\downarrow\!\mathit{DEFS}$. A browser registers rendering functions; a server registers $\mathit{httpFetch}$, $\mathit{upsert}$, $\mathit{notify}$:
\begin{align*}
(\rho\,\mathit{fact})\!:\!\mathit{render} &\to \mathit{widget} \\
(\rho\,\mathit{fact})\!:\!\mathit{httpFetch} &\to \mathit{response} \\
(\rho\,\mathit{fact})\!:\!\mathit{upsert} &\to D'
\end{align*}
Runtime functions that interact with external state execute during $\mathit{resolve}$~(\ref{eq:create}), populating cells before $\mathit{derive}$ runs. Functions that consume the representation execute after $\mathit{emit}$. This keeps $\mathit{derive}$ pure and terminating over $P$. A single $\mathit{SYSTEM}$ serves browser, server, and storage runtimes by varying $\mathit{DEFS}$, not by varying the logic.

The binding is live: when a cell's contents change via $\downarrow$, every $\rho$-application over that cell re-evaluates. This is streaming. There is no separate pub/sub layer, no event bus, no callback registry. A subscriber is a $\rho$-application that has not yet been evaluated against the current $D$. When $D$ changes, it evaluates. A sensor reading, a Kafka message, a derived inference, and a user click are all facts in $P$ or events in $E$. They enter $D$ through the same $\downarrow$ operator, and every bound $\rho$-application sees the update through the same mechanism.

\subsection{Data Federation}

A federated noun is one whose population is resolved by $\rho$ at evaluation time from an external source rather than read from a local cell. The reading ``Noun is backed by External System'' compiles to a populating function $\mathit{populate}_n$ registered in $\mathit{DEFS}$. During $\mathit{resolve}$~(\ref{eq:create}), $\rho(\mathit{populate}_n)$ fetches from the external source and produces facts:
\begin{align*}
\rho(\mathit{populate}_n)\!:\!I \;\to\; \{f_1, \ldots, f_k\} \subseteq P_{\mathit{OWA}}
\end{align*}
The fetched facts enter $P$ under OWA (absence is unknown, not false). Constraints and derivation rules evaluate over the unified $P$ without distinguishing local from federated facts. A subset constraint can span a local entity and a federated one. No schema translation is required because the fact type declared in the readings is the response schema, and JSON keys map to role bindings through the compiled schema.

The compile step constructs $\mathit{populate}_n$ from readings: the external system's URL, the noun's URI path, authentication headers, and credential references. SSRF validation rejects internal or loopback URLs at compile time, before any fetch can execute. Cell isolation (Definition~\ref{def:isolation}) ensures federated facts are cached for the request duration so that two concurrent $\mu$ applications do not re-fetch.

The approach requires no ETL, no integration middleware, and no separate query language for external data. The external system is a $\rho$-application, and the rest of the algebra does not change.

\subsection{State and Cells}

The state $D$ is a sequence of cells $\langle\mathit{CELL}, n, c\rangle$ with Backus's fetch ($\uparrow\!n\!:\!D \to c$) and store ($\downarrow\!n\!:\!\langle x, D\rangle \to D'$) operators~\cite[Sec.\,14.3]{backus78}. $\mathit{FILE}$ is a cell whose contents is~$P$. RMAP~\cite[Ch.\,10]{halpin01} determines which facts belong to which cell from the schema's uniqueness constraints: the result is a 3NF row, the complete set of facts that depend on one entity's key. Each entity is a cell. Cells may contain sub-stores~\cite[Sec.\,14.7]{backus78}.

\begin{definition}[Cell Isolation]\label{def:isolation}
For each cell $\langle\mathit{CELL}, n, c\rangle$ in $D$, at most one $\mu$ application that writes to $n$ may be in progress at any time. Concurrent $\mu$ applications over disjoint cells are permitted.
\end{definition}

\noindent RMAP assigns each entity to its own cell whose fold is independent. A single-threaded runtime satisfies Definition~\ref{def:isolation} trivially. A distributed system requires one writer per cell. Without isolation, two concurrent $\mathit{create}$ operations on the same cell may both pass $\mathit{validate}$ against stale $P$ and produce inconsistent state without an error.

A fact type $\langle\mathit{CONS}, s_1, \ldots, s_n\rangle$ is a function that produces a fact when applied to instances. Partially applied via $\mathit{bu}$ it produces a query.

Two derived forms:
\begin{align}
\mathit{Filter}(p) &\equiv \mathit{compact} \circ \alpha\,(p \to \mathit{id} ;\; \bot) \label{eq:filter-def}
\end{align}
$\alpha$ applies the condition to each element, yielding $\bot$ where $p$ does not hold. $\mathit{compact}$ removes $\bot$ elements, producing a proper subsequence. FFP systems choose their primitive set~\cite[Sec.\,11.4.4]{backus78}; $\mathit{compact}$ is an AREST primitive analogous to APL compress.

\begin{align}
\mathit{foldl}\;f\;z\;\langle\rangle &= z \notag \\
\mathit{foldl}\;f\;z\;\langle e, E'\rangle &= \mathit{foldl}\;f\;(f\;z\;e)\;E' \label{eq:foldl-def}
\end{align}
$\mathit{foldl}$ is derivable from Backus's $\mathit{while}$~\cite[Sec.\,11.2.4]{backus78}.

\noindent Every ORM\,2 constraint compiles to a restriction~(\ref{eq:restrict}) whose predicate belongs to one of two families. The \emph{cardinality} family checks whether a count falls within bounds $[k, m]$. The \emph{membership} family checks whether a tuple exists (or is absent) in a target set. Polarity is the only parameter distinguishing symmetric from asymmetric, subset from exclusion.

\subsection{Relational Algebra}

Codd's adequate $\theta_1$~\cite[Sec.\,2.2]{codd70} as FFP definitions:
\begin{align}
\pi_L(R) &\equiv \alpha\,[s_{i_1}, \ldots, s_{i_k}] : R \label{eq:proj} \\
R \mathbin{*} S &\equiv \mathit{Filter}(\mathit{eq} \circ [s_{\mathit{sh}}]) \circ \mathit{distl} \label{eq:join} \\
R_{L|_M}\!S &\equiv \mathit{Filter}(p) : R \label{eq:restrict} \\
R \cdot S &\equiv \pi_{1s}(R \mathbin{*} S) \label{eq:rcomp} \\
\gamma(R) &\equiv \mathit{Filter}(\mathit{eq} \circ [s_1, s_n]) : R \label{eq:tie}
\end{align}
\noindent where $s_{\mathit{sh}}$ selects the shared roles. $P$ is a named set of relations in normal form (ORM\,2 elementary facts are 5NF by construction~\cite[Ch.\,12]{halpin01}). Halpin shows that $\theta_1 = \{\text{projection, natural join, restriction, tie}\}$ is adequate for the named set of ORM\,2 elementary facts~\cite[Ch.\,12]{halpin01}. The combining forms that query the population are the same combining forms that evaluate constraints and derive facts. There is no separate query language.

\subsection{Execution Semantics}

$\mu$ evaluates by applicative-order $\beta$-reduction~\cite[Sec.\,13.3.1]{backus78}, repeatedly replacing $(x\!:\!y)$ with $(\rho\,x)\!:\!y$ under definitions $D$. Three constraints govern execution:
\begin{enumerate}[nosep]
\item \textbf{Reduction order:} applicative (call-by-value).
\item \textbf{Scope:} confined to the request-local partition of $D$ determined by RMAP.
\item \textbf{Derivation:} forward chaining, monotonic, evaluated to the least fixed point per request. Derivation only adds facts over finite $P$; the fixed point exists by Knaster-Tarski and is reached in $\leq |P_{\max}| - |P|$ steps. The derivation chain is recorded and available on demand.
\end{enumerate}

The system operates over facts~$(P)$, computations~$(O$ via $\rho)$, and inputs~$(I)$. Time is an input fact, not hidden state. Effects are directives in the output. Identity follows from the reference scheme; $\mathit{resolve}$~(\ref{eq:create}) applies it. The world assumption per noun~\cite[Ch.\,7]{halpin01} governs incompleteness: under CWA, a fact not in $P$ is false; under OWA, it is unknown.

\smallskip\noindent\textbf{Remark} (World assumption on populating functions). All populating functions are runtime functions registered in $\mathit{DEFS}$. A function that exhaustively enumerates its domain operates under CWA. One that cannot operates under OWA. Deontic constraints over OWA-populated facts are sound but not complete: a reported violation is genuine, but the absence of a violation is not a guarantee.

\section{Properties}

The five theorems form a chain. Each depends on the previous one and eliminates one source of failure in the path from domain knowledge to running software.

\begin{theorem}[Grammar Unambiguity]\label{thm:grammar}
Let $N$ be a declared set of object type names and $F$ a declared set of predicate readings, such that no element of $N$ contains a formal item of the grammar as a substring. Then each FORML\,2 sentence over $(N, F)$ has exactly one parse.
\end{theorem}

\begin{proof}
FORML\,2 sentences fall into three structural families, each determined by its prefix:

\smallskip\noindent
\textit{(a) Quantified constraints.} Structure:
$[\mathit{modal}] \;\; \mathit{quant}_1 \;\; n_1 \;\; r \;\; \mathit{quant}_2 \;\; n_2 \;\; [\mathit{qual}]$

\noindent where $\mathit{modal} \in \{$``It is obligatory that'', ``It is forbidden that'', ``It is permitted that'', ``It is impossible that'', ``It is possible that'', $\varepsilon\}$; $\mathit{quant}_1 \in \{$``Each'', ``For each'', ``the same'', ``No'', $\varepsilon\}$; $\mathit{quant}_2 \in \{$``at most one'', ``some'', ``exactly one'', ``at least $k$ and at most $m$'', ``more than one'', ``no'', $\varepsilon\}$. Noun positions are resolved by longest-first matching. Each structural position admits exactly one parse because (i)~modal prefixes are pairwise non-overlapping, (ii)~quantifier phrases are pairwise distinct, (iii)~no $n \in N$ contains a formal item as a substring, and (iv)~longest-first matching is deterministic. The constraint kind is determined by $(\mathit{quant}_1, \mathit{quant}_2)$.

\smallskip\noindent
\textit{(b) Conditional constraints.} The ``If\ldots then'' frame splits unambiguously on ``then''. Ring constraints are identified by all noun bases being the same type. Subset constraints by existential ``some'' in the antecedent and anaphoric ``that'' in the consequent. Derivation rules occupy the remainder.

\smallskip\noindent
\textit{(c) Multi-clause constraints.} Keyword-delimited: ``if and only if'' (EQ), ``exactly one of the following holds'' (XO), ``at most one of the following holds'' (XC), ``at least one of the following holds'' (OR). The keyword is unambiguous.

\smallskip\noindent
The three families are distinguished by their sentence prefix at position~(1). Within each family, the parse is deterministic. The reference implementation~\cite{norma} constructively demonstrates unambiguous round-trip parsing for all 17 constraint kinds.
\end{proof}

\noindent The hypothesis on $N$ is enforced by the compiler, which rejects any declared name containing a formal item of the grammar as a substring.

\begin{theorem}[Specification Equivalence]\label{thm:spec}
Let $\mathit{parse}: R \to \Phi$ and $\mathit{compile}: \Phi \to O$ be injective, with $R$ the set of FORML\,2 readings, $\Phi$ the set of FOL formulas, and $O$ the set of FFP objects. Then:
\begin{align}
\mathit{parse}^{-1} \circ\, \mathit{compile}^{-1} \circ\, \mathit{compile} \circ \mathit{parse} &= \mathit{id}_R \label{eq:spec}
\end{align}
and $\rho(\mathit{compile}(\mathit{parse}(r)))$ is the executable for every $r \in R$.
\end{theorem}

\begin{proof}
$\mathit{parse}$ is injective by Theorem~\ref{thm:grammar}. $\mathit{compile}$ is injective: each constraint kind compiles to a restriction~(\ref{eq:restrict}) with a predicate from one of two families parameterized by distinct tuples. Both have left inverses ($\mathit{compile}^{-1}$ is Halpin and Curland's verbalization algorithm~\cite{halpin06}); composing yields $\mathit{id}_R$. Executability: $\rho$ maps the compiled object to a function by~\cite[Sec.\,13.4]{backus78}.
\end{proof}

\noindent\textbf{Remark} (Surjectivity). $\mathit{compile}$ is not surjective. Runtime-registered functions occupy the complement. FORML\,2 readings cover the domain layer; registered functions cover the platform layer. Together they span $\mathit{DEFS}$.

\begin{theorem}[Completeness of State Transfer]\label{thm:complete}
Let $F_S : 2^P \to 2^P$ be the monotonic operator that applies all derivation rules in schema $S$ to a population, and let $C_S$ be the set of constraint objects in $S$. Under Definition~\ref{def:isolation}, $\mathit{create}$~(\ref{eq:create}) applied to input $I$ and state $D$ produces $\langle o, D'\rangle$ such that:
\begin{enumerate}[nosep]
\item $P' = P \cup \mathit{resolve}(S, I)$ contains all entity facts entailed by $(S, I)$;
\item $P'' = \mathit{lfp}(F_S, P')$, the least fixed point of $F_S$ above $P'$;
\item $V = \bigcup\{(\rho\,c)\!:\!P'' \mid c \in C_S\}$ collects all violations;
\item $o$ is the REST representation of $(P'', V, \mathit{links}(s))$ and $D' = D[\mathit{FILE} \mapsto P'']$ if $V$ contains no alethic violation, else $D' = D$.
\end{enumerate}
\end{theorem}

\begin{proof}
$P$ is finite. $F_S$ is monotonic. By Knaster-Tarski, $\mathit{lfp}(F_S, P')$ exists and is unique. $\mathit{validate}$ applies every $c \in C_S$ as a restriction over the fixed point. $\mathit{emit}$ constructs $o$ from $P''$, $V$, and $\mathit{links}(s)$ (Theorem~\ref{thm:hateoas}). Definition~\ref{def:isolation} ensures no concurrent write to the same cell.
\end{proof}

\begin{theorem}[HATEOAS as Projection]\label{thm:hateoas}\label{thm:nav}
All links in the representation are $\theta_1$ operations on $P$ and $S$.

\smallskip\noindent
\textit{(a) Transition links.} Let $T = \{(s_i, s_j, e_k)\} \subseteq P$ be transition facts and $s$ the current status. Then:
\begin{align}
\mathit{links}(s) &= \pi_{\mathit{event}}\!\left(\mathit{Filter}(p) : T\right) \label{eq:hateoas}
\end{align}
where $p(t) = s_{\mathit{from}}(t) \in \{s\} \cup \mathit{supertypes}(s)$. Transition graphs may contain cycles. It is obligatory that each cycle has some exit transition, a deontic constraint that ensures liveness without requiring structural acyclicity.

\smallskip\noindent
\textit{(b) Navigation links.} Let $F$ be the set of all fact types in $S$ and let $U_f$ denote the role set spanned by the UC on fact type $f$. Let $S_f = R_f \setminus U_f$ be the non-spanned roles. For a noun $n$ participating in $f$:
\begin{enumerate}
\item \textbf{Related collection.} $f$ is a related collection on $n$, filtered by $n$. Always applies.
\item \textbf{Parent/child.} If $S_f \neq \emptyset$, nouns in $S_f$ are children of nouns in $U_f$.
\item \textbf{Peer resource.} If $S_f = \emptyset$ (spanning UC), $f$ is a navigable peer resource.
\end{enumerate}
For the binary case:
\begin{align}
\mathit{children}(n) &= \pi_{\mathit{noun}(r_c)}\bigl(\mathit{Filter}( \notag \\
  &\qquad \mathit{uc}(r_c) \land \mathit{eq} \circ [\mathit{noun}(\bar{r}_c), \bar{n}]) : F_B\bigr) \label{eq:children} \\
\mathit{parent}(n) &= \pi_{\mathit{noun}(\bar{r}_c)}\bigl(\mathit{Filter}( \notag \\
  &\qquad \mathit{uc}(r_c) \land \mathit{eq} \circ [\mathit{noun}(r_c), \bar{n}]) : F_B\bigr) \label{eq:parent} \\
\mathit{nav}(e, n) &= \mathit{children}(n) \cup \mathit{parent}(n) \notag \\
  &\quad \cup\; \mathit{related}(n) \cup \mathit{peers}(n) \label{eq:nav}
\end{align}
The full link set:
\begin{align}
\mathit{links}_{\mathit{full}}(e, n, s) &= \mathit{nav}(e, n) \;\cup\; \mathit{links}(s) \label{eq:linksfull}
\end{align}
\end{theorem}

\begin{proof}
\textit{(a)} $T \subseteq P$. $\mathit{Filter}$ is restriction~(\ref{eq:restrict}); $\pi$ is projection~(\ref{eq:proj}). Both are $\theta_1$.
\textit{(b)} Each component of $\mathit{nav}$ is a projection over a restriction. The predicate composes $\mathit{eq}$ with selector and constant, all primitives from Table~\ref{tab:ffp}. All components of $\mathit{links}_{\mathit{full}}$ are $\theta_1$ operations on $S$ and $P$.
\end{proof}

\begin{theorem}[Derivability]\label{thm:derive}
Define the \emph{representation} of an entity $e$ in population $P$ under schema $S$ as:
\begin{align*}
\mathit{repr}(e, P, S) = \;&\{(\rho\,s_i)\!:\!f \mid f \in P,\, f \text{ involves } e\} \\
\cup\; &\{(\rho\,r)\!:\!P \mid r \in S_{\mathit{derivations}}\} \\
\cup\; &\{(\rho\,c)\!:\!P \mid c \in S_{\mathit{constraints}}\} \\
\cup\; &\mathit{links}_{\mathit{full}}(e, n, \mathit{status}(e, P))
\end{align*}
Then for every $v \in \mathit{repr}(e, P, S)$, there exists $f \in O$ such that $v = (\rho\,f)\!:\!P$. Equivalently: no value in the REST representation is produced outside $\rho$.
\end{theorem}

\begin{proof}
Each component is a $\rho$-application by construction. Selectors extract attributes. Derivation rules compute derived facts. Constraints produce violation sets via restriction. Links are $\theta_1$ operations (Theorem~\ref{thm:hateoas}). No value in $\mathit{repr}$ is produced outside $\rho$.
\end{proof}

\begin{corollary}[Violation Verbalization]\label{cor:verbalize}
If $(\rho\,c)\!:\!P'' \neq \emptyset$ for constraint $c$ compiled from reading $r$, the violation output is $\mathit{compile}^{-1}(c)$, recovering $r$ by Theorem~\ref{thm:spec}. The error message is the original reading.
\end{corollary}

\noindent Halpin and Curland~\cite{halpin06} give the automated verbalization algorithm. Composed $\theta_1$ operations verbalize via FORML\,2 qualified readings, which parse unambiguously by Theorem~\ref{thm:grammar}.

\begin{corollary}[Deletion as Terminal State]\label{cor:delete}
A state with no outgoing transitions produces $\mathit{links}(s) = \emptyset$ by Theorem~\ref{thm:hateoas}. The entity is excluded from query results by $\mathit{Filter}(p_{\mathit{live}}) : P$.
\end{corollary}

\begin{corollary}[Closure Under Self-Modification]\label{cor:closure}
After applying $\mathit{SYSTEM}$ with operation $\mathit{compile} \circ \mathit{parse}$ to readings $R'$, Theorems~\ref{thm:grammar}--\ref{thm:derive} hold for all subsequent applications over $D'$.
\end{corollary}

\begin{proof}
Theorem~\ref{thm:grammar} depends on the grammar, not on $D$. Theorem~\ref{thm:spec} depends on injective stateless functions. New definitions enter $\mathit{DEFS}$ via $\downarrow\!\mathit{DEFS}$. Theorems~\ref{thm:complete}--\ref{thm:derive} operate over $P$ and $S$, both of which include the new content. No theorem's proof depends on $D$ being fixed at initialization.
\end{proof}

\smallskip\noindent\textbf{Remark} (Self-hosting). This is the property the formalism turns on: AREST ingests schemas that extend its own schema, and every theorem in the chain continues to apply. The system is not merely specified by readings; it is extended by them.

\smallskip\noindent\textbf{Remark} (Migration). The compile op is itself a $\mathit{SYSTEM}$ application and subject to $\mathit{validate}$~(\ref{eq:create}). If an ingested schema introduces an alethic constraint that $P$ violates, the ingestion is rejected ($D' = D$). Migration readings that transform $P$ must therefore land in the same ingestion --- as derivation rules, transition triggers, or deontic rather than alethic constraints --- or as a staged sequence whose intermediate readings produce compliant $P$ before the stricter constraint is introduced.

\begin{corollary}[Constraint Consensus]\label{cor:consensus}
For two peers sharing population $P$, one proposes event $e$ and the other applies $\mathit{validate}$~(\ref{eq:create}) to $P \cup \mathit{resolve}(S, e)$. The event is accepted iff $V$ contains no alethic violation. The constraint set $C_S$ is the consensus predicate.
\end{corollary}

\noindent For $n > 2$ peers, a total ordering over the shared event log is additionally required. Safety follows from deterministic replay; liveness from the single-writer cell (Definition~\ref{def:isolation}).

\section{Distributed Evaluation}

Each cell stores the 3NF row. The event stream is demultiplexed into per-cell streams by RMAP:
\begin{align}
E_n &= \mathit{Filter}(\mathit{eq} \circ [\mathit{RMAP}, \bar{n}]) : E \label{eq:demux}
\end{align}
Each cell folds its stream independently:
\begin{align}
D_n' &= \mathit{foldl}\;\mu_n\;D_n\;E_n \label{eq:cellfold}
\end{align}
where $\mu_n$ is $\mathit{SYSTEM}$ restricted to cell $n$. Cross-cell queries read committed state:
\begin{align}
P &= \textstyle\bigcup_n \uparrow\!\mathit{FILE}\!:\!D_n \label{eq:pop}
\end{align}
Adding cells to the cluster scales $\mathit{SYSTEM}$ horizontally without changing its definition.

A browser runtime is a node in the same model. Cells replicated locally fold events with zero latency, syncing asynchronously. Because the fold is deterministic, two nodes folding the same events reach the same state. Local constraints (cardinality, value) evaluate immediately. Cross-cell constraints (subset, external uniqueness) evaluate against committed state after sync. Local evaluation is optimistic feedback, not a security boundary.

For anonymous peers, events carry cryptographic signatures for identity, schema modifications require authorization (a deontic constraint), and every peer validates independently. The hash chain provides ordering, constraints provide validity, signatures provide identity, and authorization provides permission, all composed from the existing formalism.

\section{Middleware Elimination}

Traditional middleware (authentication, validation, rate limiting, transformation) is either a restriction over $P$, a composition, or a fact in $P$. Authorization is a derivation: ``User accesses Domain iff User belongs to Organization that manages Domain'' compiles to $\langle\mathit{COMP}, f, g\rangle$ whose $\rho$-image computes the access predicate over $P$. External service calls, database persistence, and UI rendering are runtime functions registered into $\mathit{DEFS}$ and applied via $\rho$. Theorem~\ref{thm:derive} leaves no room for a computation layer outside $\rho$.

\section{Conclusion}

The five theorems form a dependency chain from unambiguous grammar to full derivability, each eliminating one source of failure between domain knowledge and running application. Self-modification preserves all five (Corollary~\ref{cor:closure}). Fielding's remaining architectural constraints follow: statelessness holds because each application of $\mathit{SYSTEM}$ is a pure transition on $D$; cacheability because the representation is a deterministic function of $P$ and $S$ (Theorem~\ref{thm:derive}); layered system because cells are location-independent; code-on-demand because self-modification via ingesting readings preserves the theorems.

The five properties compose into a guarantee for automated agents. An LLM writing FORML\,2 readings produces unambiguous specifications (Theorem~\ref{thm:grammar}) that are the executables (Theorem~\ref{thm:spec}). An LLM consuming the API reads values that are $\rho$-applications over $P$ (Theorem~\ref{thm:derive}) and discovers every valid action via HATEOAS links (Theorem~\ref{thm:hateoas}). There are no undocumented endpoints, no hidden middleware, no informal contracts to hallucinate through. The forward chainer evaluates derivation rules to the least fixed point (Theorem~\ref{thm:complete}), and the LLM reads conclusions, not premises. The logical chain was evaluated deterministically by the engine. When the chain itself is needed, the derivation trace is available on demand.

The algebraic laws of FP~\cite[Sec.\,12.2]{backus78} are equations between functional forms. Since FFP objects represent the same forms via $\rho$, an FFP object can be transformed to an equivalent object by applying these laws. Ahead-of-time compilation from objects to optimized evaluation trees is one such transformation.

Common operational concerns map directly to the formalism: versioning is the event stream, migration is ingestion of new readings, backward compatibility is preserved by construction, and auditability is the derivation trace.

The gap between domain knowledge and running software is the gap where requirements drift, implementations diverge, and production breaks. AREST eliminates the gap by identifying what was never separate to begin with.
